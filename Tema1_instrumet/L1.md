
Лекция 2.1. Современный инструментарий разработчика: VS Code, терминал, жизненный цикл кода цифрами
Цель лекции:
Настроить рабочее место, перестать бояться чёрного окна терминала и понять, что код — это не просто файлы, а живой процесс с чёткими этапами.

Важно: всё, что показывается на лекции, вы обязаны сделать руками на практике. Не просто посмотреть, а ввести каждую команду, нажать каждую кнопку.

1. VS Code — не просто редактор, а среда управления кодом
VS Code — это главный инструмент. Он лёгкий, но мощный. Наша задача: превратить его в «пульт управления разработкой».

1.1. Первые 3 шага после установки
Открыть папку проекта — именно так, а не просто файл.
Файл → Открыть папку (или Ctrl+K Ctrl+O).
Вся работа строится вокруг папки.

Включить боковую панель (иконка «две полоски» слева).
Там будет: Проводник файлов, Поиск, Управление версиями (Git), Расширения.

Открыть встроенный терминал — это ключевой момент.
Терминал → Новый терминал или Ctrl+`.

Шаг 1. Замысел
Сделаем простую страницу с кнопкой, которая считает клики.

Шаг 2. Пишем код
В VS Code создайте папку counter, внутри файл index.html.

Вставьте код:

    <script>
        let count = 0;
        const btn = document.getElementById('btn');
        const span = document.getElementById('count');

        btn.onclick = function() {
            count = count + 1;
            span.innerText = count;
        }
    </script>

</html>
Сохраните (Ctrl+S).

Способ 2 (через терминал):

Шаг 4. Отладка (нарочно сломаем)
Удалите одну скобку в строке span.innerText = count; (станет span.innerText = count; без точки с запятой или с ошибкой).
Сохраните, обновите страницу, нажмите кнопку — ничего не работает.
Откройте консоль браузера (F12 → вкладка Console) — увидите красную ошибку.
Исправьте обратно.

Шаг 5. Сохраняем версию (имитация Git)
Пока просто создайте папку versions и скопируйте туда работающий файл с именем index_v1.html. Это грубая, но понятная замена Git на первом занятии.

Практическая работа (ПЗ 2.1). Ваша первая самостоятельная разработка
Что нужно сделать (выполняется на том же занятии или дома):

Выполнить команды из п. 2.3 — создать папку и файл app.js, выполнить его через node app.js.

Создать проект «Калькулятор»

HTML-страница с двумя полями ввода (числа), кнопками «+», «-», «*», «/» и полем для результата.

Написать JavaScript, который выполняет действие при нажатии кнопки.

Проверить работу, найти и исправить 2 намеренные ошибки (например, забыли преобразовать строку в число).

Сохранить итоговую рабочую версию в отдельную папку final.
