ПР 2.1. Настройка VS Code для Python-проекта: расширения, отладчик, линтеры. Создание первого Python-модуля
Цель работы:
Настроить VS Code как профессиональную среду для Python, написать и запустить свой первый многофайловый Python-модуль, научиться отлаживать код с помощью штатного отладчика.

Требования к выполнению:

Работа выполняется индивидуально.

Каждый шаг фиксируется скриншотом или коротким пояснением в отчёте.

В конце работы все настройки (расширения, конфиги) должны быть сохранены в папке проекта.

Время выполнения: ~90 минут.

Часть 1. Настройка VS Code под Python (30 минут)
Шаг 1. Установка расширений (обязательный минимум)
Откройте VS Code → нажмите Ctrl+Shift+X → установите по очереди:

Расширение	Назначение
Python (Microsoft)	Всё для Python: IntelliSense, отладка, линтеры, тесты
Pylint	Проверка стиля кода (линтер)
Python Docstring Generator	Быстрое создание документации к функциям
Path Intellisense	Автодополнение путей к файлам
После установки: перезагрузите VS Code (Ctrl+Shift+P → Reload Window).

Шаг 2. Выбор интерпретатора Python
Откройте любую папку (создайте новую python_prj_01).

Ctrl+Shift+P → введите Python: Select Interpreter.

Выберите версию Python 3.8+ (если нет — установите с python.org).

Проверка: в левом нижнем углу VS Code появится версия Python (например, Python 3.10.11 64-bit).

Шаг 3. Настройка линтера (Pylint)
Линтер подсвечивает ошибки стиля и синтаксиса прямо во время набора.

Ctrl+Shift+P → Python: Select Linter → выберите Pylint.

Проверка: создайте файл test.py и нарочно напишите:

python
x=  5
print x
Pylint должен подчеркнуть красным:

пробелы вокруг = (PEP8 требует x = 5)

отсутствие скобок у print (в Python 3 нужно print(x)).

Шаг 4. Создание конфигурации для отладчика
В VS Code отладчик работает через файл launch.json.

Перейдите на вкладку «Запуск и отладка» (иконка ▶ с жучком слева).

Нажмите создать файл launch.json → выберите Python.

В открывшемся файле оставьте только конфигурацию для текущего файла:

json
{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "Python: текущий файл",
            "type": "python",
            "request": "launch",
            "program": "${file}",
            "console": "integratedTerminal"
        }
    ]
}
Сохраните файл (Ctrl+S).

Что мы получили: теперь можно нажать F5 и запустить любой Python-файл под отладчиком.

Часть 2. Создание первого Python-модуля (30 минут)
Задание: модуль для работы с геометрическими фигурами
Мы создадим свой модуль geometry.py, который содержит функции для вычисления площади и периметра фигур. Затем напишем программу, которая его использует.

Шаг 1. Структура проекта
Внутри папки python_prj_01 создайте:

text
python_prj_01/
├── geometry.py      # наш модуль
├── main.py          # программа, использующая модуль
└── .vscode/         # папка с настройками (launch.json уже здесь)
Шаг 2. Код модуля geometry.py
python
"""
Модуль для расчёта параметров геометрических фигур.
Содержит функции: rectangle_area, rectangle_perimeter, circle_area, circle_circumference.
"""

def rectangle_area(a, b):
    """Вычисляет площадь прямоугольника."""
    return a * b

def rectangle_perimeter(a, b):
    """Вычисляет периметр прямоугольника."""
    return 2 * (a + b)

def circle_area(r):
    """Вычисляет площадь круга."""
    pi = 3.14159
    return pi * r * r

def circle_circumference(r):
    """Вычисляет длину окружности."""
    pi = 3.14159
    return 2 * pi * r
Что сделали: создали самодокументируемый код (каждая функция имеет докстринг).

Шаг 3. Программа main.py, использующая модуль
python
# Импортируем функции из нашего модуля
from geometry import rectangle_area, circle_area

# Ввод данных от пользователя
print("Расчёт площади фигуры")
print("1 - прямоугольник, 2 - круг")
choice = input("Выберите фигуру: ")

if choice == "1":
    a = float(input("Длина: "))
    b = float(input("Ширина: "))
    result = rectangle_area(a, b)
    print(f"Площадь прямоугольника: {result}")
elif choice == "2":
    r = float(input("Радиус: "))
    result = circle_area(r)
    print(f"Площадь круга: {result}")
else:
    print("Неверный выбор")
Шаг 4. Запуск и проверка
Без отладки:
python main.py в терминале VS Code.

С отладкой (важно!):

Поставьте точку останова — кликните слева от номера строки в main.py (например, на result = rectangle_area(a, b)). Появится красный кружок.

Нажмите F5.

Программа остановится на точке останова. Посмотрите значения переменных a, b в панели слева.

Нажмите F10 (шаг с обходом) — выполнится вызов функции и вы увидите результат.

Нажмите F5 для продолжения.

Часть 3. Линтер и отладчик в действии (30 минут)
Задание 1. Исправляем ошибки с помощью линтера
Напишите в geometry.py дополнительную функцию с нарочными ошибками:

python
def triangle_area(a,b,c):  # ошибка: не используется с
    p=(a+b+c)/2
    return sqrt(p*(p-a)*(p-b)*(p-c))  # ошибка: sqrt не импортирован
Pylint сразу подчеркнёт:

неиспользуемую переменную c

неизвестное имя sqrt

Исправление: добавьте from math import sqrt в начало файла и удалите c из параметров.

Задание 2. Отладка с условиями (условная точка останова)
В main.py после ввода чисел добавьте проверку:

python
if a <= 0 or b <= 0:
    print("Ошибка: стороны должны быть положительными")
    exit()
Поставьте точку останова на эту проверку.

Нажмите правой кнопкой на красный кружок → «Изменить условие...» → введите a <= 0 or b <= 0.

Запустите отладку (F5) и введите отрицательное значение. Отладчик остановится только при ошибке, а не всегда.
