Пример проверочного скрипта для варианта 16 (Симуляция)
python
import simpy
import numpy as np

def car(env):
    while True:
        print(f"Парковка в {env.now}")
        parking_duration = np.random.exponential(5)
        yield env.timeout(parking_duration)
        print(f"Выезд в {env.now}")

env = simpy.Environment()
env.process(car(env))
env.run(until=20)
print("Simpy работает!")

# === СОЗДАНИЕ ОКРУЖЕНИЯ ===
python -m venv venv                    # создать
source venv/bin/activate               # активировать (macOS/Linux)
venv\Scripts\activate                  # активировать (Windows)
deactivate                             # выйти из окружения

# === УПРАВЛЕНИЕ ПАКЕТАМИ ===
pip install <package>                  # установить
pip install <package>==1.2.3           # установить конкретную версию
pip install --upgrade <package>        # обновить
pip uninstall <package>                # удалить
pip list                               # посмотреть все
pip list --outdated                    # устаревшие пакеты
pip show <package>                     # информация о пакете

# === ЗАВИСИМОСТИ ===
pip freeze > requirements.txt          # сохранить
pip install -r requirements.txt        # восстановить
pip check                              # проверить конфликты
Методические рекомендации для преподавателя
Раздаточный материал: распечатайте карточки с вариантами (по одной на студента). Номер варианта студент получает случайно (по списку или вытягивает).

Время выполнения: 60-90 минут на вариант (зависит от уровня).

Критерии оценки по варианту 0 (эталон):

5 баллов: все шаги выполнены, проверочный скрипт работает после восстановления

4 балла: скрипт работает, но есть ошибки в .gitignore или requirements.txt содержит лишние строки

3 балла: окружение создано, но не восстановлено

2 балла и ниже: задание не выполнено

Распространённые ошибки (на что обращать внимание):

Забыли активировать venv перед pip install

Закоммитили папку venv (не создали .gitignore)

requirements.txt создали без активации (содержит глобальные пакеты)

Не проверили работу скрипта после восстановления
